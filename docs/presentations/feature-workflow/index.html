What's an agent team?

A team is a group of independent Claude Code sessions working together. One is the lead that coordinates; the others are teammates with specialized roles. Each runs in its own context and can message the others directly.

A subagent is a one-shot worker that returns to its parent. A teammate is a long-running peer that negotiates with the others until the work is done.

Independent contexts Each teammate keeps its own conversation. Specialized focus stays uncontaminated by other roles.
Peer messaging Teammates address each other by name. Subagents can only report back up to their parent.
Adversarial separation One teammate ships, another independently challenges. Bias is built out of the workflow.

  • RESUME — picks up an in-progress epic from epic-state.json
  • NEW — validates a fresh spec file and starts planning
  • SCAN — checks specs/epic-* for unfinished epics
  • NATURAL_LANGUAGE — converts prose into a spec, then continues
  • Checks for: problem statement, testable ACs, scope boundaries, error handling

  • Delegates complex specs to the spec-validator subagent
  • Detects project language from config files
  • Creates specs/epic-{name}/ with epic-state.json
  • Spawns 2–3 code-explorer subagents in parallel (features, architecture, testing)
  • Initialises mocks-registry.json
